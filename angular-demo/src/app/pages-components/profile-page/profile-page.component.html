Pages vs Components in Angular

In Angular, there is no technical difference between a "Page" and a "Component". They are both just @Component classes!

The Conceptual Difference

A "Page" (Smart Component): Is attached to a Route (like this one!). It manages state, fetches data, and handles layout.

A "Component" (Dumb Component): Is reusable UI. It doesn't know about the URL. It just receives data via input() and emits events via output().
