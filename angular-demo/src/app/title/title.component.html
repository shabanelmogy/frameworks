Variables and Inputs in Angular

Angular provides several ways to declare variables and receive data from parent components. Here is how they compare:

1. Normal Variables

Standard TypeScript class properties. They are not deeply reactive on their own outside of Angular's Zone.js change detection cycle. You use standard interpolation {{ property }} to display them.

Title: {{ title }}

Description: {{ description }}

2. Signal Variables

Introduced in Angular 16+, signals are reactive wrappers around a value. They tell Angular exactly when they change, making your app extremely fast. In the template, you MUST call them as a function: {{ signalExample() }}.

Value: {{ signalExample() }}

3. Optional Signal Inputs

The modern replacement for @Input(). This acts exactly like a [Parameter] in Blazor. It receives data from the parent component. Because it's a signal, it's read-only from inside this component, and reactive.

Value provided by Parent: {{ optionalInput() }}

4. Required Signal Inputs

Like [Parameter, EditorRequired] in Blazor. It enforces that the parent component must pass a value, otherwise Angular throws a runtime error. It cannot have a default value.

Required Value from Parent: {{ requiredInput() }}
